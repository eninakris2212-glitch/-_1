33.1. Итоговый проект по автоматизации тестирования (PJ-04) 

Объект тестирования: Ростелеком (https://b2c.passport.rt.ru/ 

Ссылка на чек-лист, тест-кейсы, баги https://docs.google.com/spreadsheets/d/1X7SP7u5th2lQTjbiwIQtg5CNp4eBM82yF1-rni1W8bw/edit?usp=sharing  

О проекте 

Проект представляет собой набор автоматизированных тестов для проверки функциональности и безопасности системы авторизации на сайте https://b2c.passport.rt.ru. Тесты разработаны с использованием Python, библиотеки Selenium и фреймворка Pytest. 

Что проверяют тестовые сценарии 

Автоматизированные тесты охватывают следующие ключевые сценарии: 

Авторизация: успешная и неуспешная авторизация по разным данным (почта, телефон, логин), работа кнопки "Запомнить меня", переключение методов авторизации 

Восстановление пароля: по SMS, почте, логину, временная блокировка при частых попытках 

Регистрация: проверка валидации данных, предотвращение дубликатов почты и телефона 

Безопасность: работа капчи, управление сессиями, защита от утечек данных через URL 

Интерфейс: работа кнопки "Показать пароль", поддержка разных разрешений экрана, совместимость с браузерами

Инструкции по запуску автотестов 

Предварительные требования 

Установленный Python 3.x 

Установленный браузер Chrome 

Установленный драйвер ChromeDriver, подходящий для вашей версии Chrome 

Установка зависимостей 

Создайте виртуальное окружение и установите необходимые пакеты: 

python -m venv venv 
source venv/bin/activate # Linux/macOS 
venv\Scripts\activate # Windows 
pip install -r requirements.txt 

Запуск тестов 

Для запуска всех тестов выполните команду: 

pytest -v --driver Chrome --driver-path=/path/to/chromedriver 

где /path/to/chromedriver - путь к вашему драйверу ChromeDriver.
